Data sources & settings
Live data is fetched from your browser. JPL/CNEOS and NeoWs support CORS. If opened from a file:// path some browsers block requests \u2014 serve over http for live data.
Use backend proxy (recommended)routes JPL/NeoWs through your server \u2014 fixes CORS, caches, hides the key
JPL/CNEOS (ssd-api) often blocks direct browser requests (CORS). Paste a CORS proxy prefix here, or route through the backend. The proper fix for an institutional deployment is the server-side proxy.

Alert thresholds (Act)
When an object crosses a threshold it enters the alert console. Browser notifications and a webhook dispatch are optional; real email/Telegram/SMS relaying and multi-user acknowledgment require the server-side backend (channel secrets must stay on the server).
